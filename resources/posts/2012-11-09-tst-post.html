As of a few hours ago, the bigger part of my afmongodb driver has been merged to the official syslog-ng 3.3 tree!

A few things remain to be done, however:

  • Authentication doesn't work. This is due to the underlying client library not supporting it yet. The code is still in the driver, commented out, and once libmongo-client gains authentication support, it can be re-enabled.
  • Dynamic variable support. While this is one of the major selling points of the driver, this functionality will be added as a generic framework, which all drivers can use, if they wish to. The plan is to get the framework coded and merged, then the driver updated to support it.

Apart from the above two, of which, the second will be a tricky thing to implement, the driver is complete and as far as my tests show, stable.
